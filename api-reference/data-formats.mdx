# Data Formats

This page explains the data formats, schemas, and conventions used throughout the Influship API.

## Core Data Types

### Engagement Rates
All engagement rates are expressed as decimal values between 0.0 and 1.0:

```json
{
  "engagement_rate": 0.05,  // 5% engagement rate
  "like_rate": 0.12,        // 12% like rate
  "comment_rate": 0.02      // 2% comment rate
}
```

### Match Scores
Match scores range from 0.0 to 1.0, where:
- `0.0` = No match
- `0.5` = Moderate match  
- `1.0` = Perfect match

```json
{
  "match_score": 0.87,
  "decision": "good",
  "confidence": 0.92
}
```

### Timestamps
All timestamps use ISO 8601 format with UTC timezone:

```json
{
  "created_at": "2025-01-15T10:30:00Z",
  "updated_at": "2025-01-15T14:45:30Z"
}
```

## Target Identifiers

The API uses platform-prefixed identifiers for different target types:

### Creator IDs
ULID format with `cr_` prefix:
```json
{
  "id": "cr_01HZ8K9M2N3P4Q5R6S7T8U9V0W"
}
```

### Profile IDs
Platform and username combination:
```json
{
  "id": "instagram:username",
  "id": "tiktok:creator_handle"
}
```

### Post IDs
Platform and post identifier:
```json
{
  "id": "instagram:post_id",
  "id": "tiktok:video_id"
}
```

```json
{
  "overall_score": 0.85,
  "category": "safe",
  "risks": [],
  "confidence": 0.92,
  "breakdown": {
    "content_appropriateness": 0.90,
    "brand_alignment": 0.80,
    "audience_safety": 0.85
  }

```

### Content Analysis
```json
{
  "topics": ["fitness", "nutrition", "lifestyle"],
  "sentiment": "positive",
  "call_to_action": true,
  "product_mentions": ["protein_powder", "gym_equipment"],
```

## Error Response Format

All errors follow a consistent format:

```json
{
  "error": {
    "code": "validation_error",
    "message": "The request body is invalid",
    "details": {
      "field": "query",
      "issue": "Query must be at least 3 characters long"
    },
    "request_id": "req_123456789"
  }
}
```

## Pagination Format

List responses include pagination metadata:

```json
{
  "items": [...],
  "filtered_total": 150,
  "has_more": true,
  "next_cursor": "eyJvZmZzZXQiOjIwfQ==",
  "pagination": {
    "limit": 20,
    "offset": 0,
    "total": 150
  }
}
```

## Job Response Format

Async operations return job information:

```json
{
  "job_id": "job_123456789",
  "status": "queued",
  "created_at": "2025-01-15T10:30:00Z",
  "estimated_completion": "2025-01-15T10:35:00Z"
}
```

## Webhook Payload Format

Webhook notifications include job status updates:

```json
{
  "event": "job.completed",
  "job_id": "job_123456789",
  "status": "completed",
  "result": {
    "targets": [...]
  },
  "timestamp": "2025-01-15T10:35:00Z"
}
```
