# Data Formats

This page explains the data formats and field types used throughout the Influship API.

## Common Field Types

### Engagement Rate

- **Type**: `number` (float)
- **Range**: 0.0 to 1.0
- **Example**: `0.045` (4.5% engagement rate)

### Follower Count

- **Type**: `integer`
- **Example**: `125000`

### Timestamps

- **Type**: `string`
- **Format**: ISO 8601
- **Example**: `"2024-01-15T10:30:00Z"`

### Creator ID

- **Type**: `string`
- **Format**: UUID
- **Example**: `"00000000-0000-0000-0000-000000000000"`

### Platform

- **Type**: `string`
- **Values**: `"instagram"` (more platforms coming soon)

## Response Formats

### Success Response

```json
{
  "items": [...],
  "filtered_total": 45,
  "has_more": true,
  "next_cursor": "eyJvZmZzZXQiOjEwfQ=="
}
```

### Error Response

```json
{
  "error": {
    "code": "invalid_request",
    "message": "The request is missing required parameters",
    "details": {
      "missing_fields": ["query"]
    }
  }
}
```

## Pagination

### Cursor-based Pagination

- Use `next_cursor` from previous response
- Include `cursor` parameter in next request
- `has_more` indicates if more results exist

### Limit Parameter

- **Default**: 25
- **Maximum**: 100
- **Minimum**: 1

## Rate Limit Headers

All responses include rate limit information:

- `X-RateLimit-Limit`: Maximum requests per hour
- `X-RateLimit-Remaining`: Requests remaining this hour
- `X-RateLimit-Reset`: Unix timestamp when limits reset
