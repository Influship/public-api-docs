# API Reference

Comprehensive documentation for the Influship API. Build powerful influencer marketing applications with our AI-powered search, analysis, and matching capabilities.

## Base URL

All API requests are made to:

```
https://api.influship.com
```

## Authentication

All endpoints (except `/health`) require an API key passed in the `X-API-Key` header:

```bash
curl -H "X-API-Key: YOUR_API_KEY" https://api.influship.com/v1/search
```

<Warning>
**Keep your API key secure!** Never expose your API key in client-side code or public repositories. Treat it like a password.
</Warning>

## Pagination

Endpoints returning lists use **cursor-based pagination** for efficient data traversal:

### How It Works

1. **Request with limit** - Set how many results you want per page (default: 25, max: 100)
2. **Check `has_more`** - Response indicates if more results exist
3. **Use `next_cursor`** - Pass this value as `cursor` parameter for the next page
4. **Track total** - Use `filtered_total` to show total matching results

<CodeGroup>

```bash First Page
curl "https://api.influship.com/v1/search?limit=25" \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query": "fitness influencers"}'
  
# Response includes:
# {
#   "items": [...],
#   "filtered_total": 247,
#   "has_more": true,
#   "next_cursor": "eyJpZCI6MjV9"
# }
```

```bash Next Page
curl "https://api.influship.com/v1/search?limit=25&cursor=eyJpZCI6MjV9" \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query": "fitness influencers"}'
```

</CodeGroup>

<Note>
**Cursors are opaque** - They're base64-encoded strings containing pagination state. Never try to construct or decode them manually. Always use the exact `next_cursor` value from the previous response.
</Note>

### Pagination Fields

<ParamField path="limit" type="integer" default="25">
  Number of results per page. Min: 1, Max: 100
</ParamField>

<ParamField path="cursor" type="string">
  Opaque pagination cursor from previous response's `next_cursor`
</ParamField>

<ParamField path="filtered_total" type="integer">
  Total matching results before pagination. Use this to show "25 of 1,247 results"
</ParamField>

<ParamField path="has_more" type="boolean">
  Whether additional pages exist. If `true`, use `next_cursor` to fetch more
</ParamField>

## Response Format

### Success Response

All successful responses follow this structure:

```json
{
  "items": [...],          // Or other data field
  "filtered_total": 247,   // Total matching results (for paginated endpoints)
  "has_more": true,        // More pages available
  "next_cursor": "..."     // Cursor for next page
}
```

### Error Response

All errors include a structured error object with actionable details:

```json
{
  "error": {
    "code": "validation_error",
    "message": "Request validation failed",
    "details": {
      "fields": ["query"],
      "context": "query must be between 1-500 characters"
    },
    "request_id": "req_8a7b9c0d1e2f3g4h"
  },
  "timestamp": "2024-01-15T10:30:00.000Z"
}
```

<AccordionGroup>
<Accordion title="Common Error Codes">
  - `validation_error` - Request validation failed (400)
  - `unauthorized` - Invalid or missing API key (401)
  - `not_found` - Resource not found (404)
  - `rate_limit_exceeded` - Too many requests (429)
  - `internal_error` - Server error (500)
</Accordion>

<Accordion title="Endpoint-Specific Errors">
  - `seed_not_found` - Lookalike seed creator not found (404)
  - `invalid_seeds` - Invalid seed data provided (400)
  - `brand_safety_failed` - Brand safety analysis failed (500)
  - `post_analysis_failed` - Post analysis failed (500)
</Accordion>
</AccordionGroup>

## Response Headers

Every API response includes metadata headers for tracking usage and limits:

### Billing & Usage Headers

<ResponseField name="X-Request-Id" type="string">
  Unique identifier for this request. Include in support tickets for faster troubleshooting.
  
  **Example**: `req_8a7b9c0d1e2f3g4h`
</ResponseField>

<ResponseField name="X-Credits-Charged" type="string">
  Exact number of credits consumed by this request
  
  **Example**: `4.15`
</ResponseField>

<ResponseField name="X-Credits-Features" type="string">
  Comma-separated list of features that impacted billing
  
  **Example**: `detailed_mode,ai_analysis`
</ResponseField>

<ResponseField name="X-Billing-Plan" type="string">
  Your current billing plan identifier
  
  **Example**: `pro`
</ResponseField>

### Rate Limit Headers

<ResponseField name="RateLimit-Limit" type="string">
  Maximum requests allowed in the current time window
  
  **Example**: `10000`
</ResponseField>

<ResponseField name="RateLimit-Remaining" type="string">
  Requests remaining in current window
  
  **Example**: `9995`
</ResponseField>

<ResponseField name="RateLimit-Reset" type="string">
  Unix timestamp (seconds) when rate limit resets
  
  **Example**: `1704124800`
</ResponseField>

## Common Patterns

Understanding these workflows helps you build efficient integrations.

### 1. Discovery â†’ Enrichment Flow

<Steps>

<Step title="Get Creator Info">
  Use `creator_profile_id` with `/v1/creators`
  
  **Result**: Person-level data (name, bio, avatar)

</Steps>

### 2. Search â†’ Analysis Flow

<Steps>
<Step title="Search">
  Use `/v1/search` with natural language query
  
  **Result**: Matching creators with AI recommendations
</Step>

<Step title="Analyze">
  Use returned creator IDs with brand safety or match endpoints
  
  **Result**: Suitability scores and risk assessment
</Step>

<Step title="Deep Dive">

  **Result**: Detailed post data with optional AI features
</Step>
</Steps>

<Steps>

## Data Concepts

### Creator vs Profile Data

Understanding the difference is key to using the API effectively:

<CardGroup cols={2}>

<Card title="Creator Data" icon="user">
  **Person-level information**
  
  - Name, bio, avatar
  - Cross-platform identity
  - Aggregated information
  
  **Endpoint**: `/v1/creators`
</Card>

<Card title="Profile Data" icon="instagram">
  **Platform-specific metrics**
  
  - Follower counts, engagement
  - Verified status
  - Platform username & handle
  
  **Endpoints**: `/v1/profiles`, `/v1/creators/{id}/profiles`
</Card>

</CardGroup>

### Response Modes

Control the level of detail in API responses:

<ParamField path="mode" type="string" default="lite">
  **`lite`** - Basic fields only (faster, cheaper)
  
  **`detailed`** - Extended metrics and analytics (+0.05 credits per social account)
</ParamField>

<Accordion title="Lite Mode Fields">
  - Basic profile info (username, display name, bio)
  - Follower count, engagement rate
  - Verification status
  - Avatar and external URLs
</Accordion>

<Accordion title="Detailed Mode Additional Fields">
  - Average comments & video views
  - Monthly follower growth rate
  - Platform category
  - Following count, media count
  - Highlight reel count (Instagram)
</Accordion>

## Quick Start

<CodeGroup>

```bash cURL
# Search for fitness influencers
curl -X POST https://api.influship.com/v1/search \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "fitness influencers with 100k+ followers",
    "mode": "detailed",
    "filters": {
      "platform_filters": [{
        "platform": "instagram",
        "min_followers": 100000,
        "min_engagement_rate": 3
      }]
    }
  }'
```

```javascript JavaScript
const response = await fetch('https://api.influship.com/v1/search', {
  method: 'POST',
  headers: {
    'X-API-Key': process.env.INFLUSHIP_API_KEY,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    query: 'fitness influencers with 100k+ followers',
    mode: 'detailed'
  })
});

const data = await response.json();
console.log(`Found ${data.filtered_total} creators`);
```

```python Python
import requests

response = requests.post(
    'https://api.influship.com/v1/search',
    headers={
        'X-API-Key': 'YOUR_API_KEY',
        'Content-Type': 'application/json'
    },
    json={
        'query': 'fitness influencers with 100k+ followers',
        'mode': 'detailed'
    }
)

data = response.json()
print(f"Found {data['filtered_total']} creators")
```

</CodeGroup>
