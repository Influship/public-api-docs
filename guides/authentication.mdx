# Authentication

The Influship API uses API key authentication. All requests must include your API key in the `X-API-Key` header.

## Getting Your API Key

Reach out to [elliot@influship.com](mailto:elliot@influship.com) to get an API key

:::info Self-Serve Dashboard Coming Soon
A self-serve API keys dashboard is coming soon, which will allow you to manage your API keys, view usage statistics, and monitor your rate limits directly from the web interface.
:::

## Using Your API Key

Include your API key in the `X-API-Key` header for all requests:

<CodeGroup>
```bash cURL
curl -H "X-API-Key: YOUR_API_KEY" \
  https://api.influship.com/health
```

```javascript Node.js
const response = await fetch('https://api.influship.com/health', {
  headers: {
    'X-API-Key': process.env.INFLUSHIP_API_KEY,
  },
});
```

```python Python
import requests

response = requests.get(
  'https://api.influship.com/health',
  headers={'X-API-Key': 'YOUR_API_KEY'}
)
```

</CodeGroup>

<ResponseExample>

## Rate Limiting

The API includes rate limiting headers in all responses to help you monitor your usage:

- `RateLimit-Limit`: Your requests per minute limit
- `RateLimit-Remaining`: Remaining requests in the current minute
- `RateLimit-Reset`: Unix timestamp when the rate limit resets
- `Retry-After`: Seconds to wait before retrying (only present on 429 responses)

## Best Practices

- Store your API key as an environment variable
- Never commit API keys to version control
- Use different keys for development and production
- Rotate keys regularly for security
- Monitor rate limit headers to avoid hitting limits
- Implement exponential backoff when receiving 429 responses
